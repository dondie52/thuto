**Anti-Patterns Verdict**

It does not look like a throwaway AI app. It has real product thinking: compare limits, URL-restored filters, local saved state, eligibility caveats, and useful data workflows. The risk is different: it looks like a capable prototype that has accumulated cards, pills, small uppercase labels, blur panels, and repeated grids faster than it has accumulated a strict design system.

The deterministic scan found 2 warnings: `gray-on-color` in `src/components/PredictorGradeSection.jsx:115`, and overused `Fraunces` in `src/index.css:1`. I also manually found repeated card grids, glass-like panels used decoratively, hero metric cards, excessive tiny labels, and mobile navigation density.

**What's Working**

- The product has strong task coverage: predictor, fit finder, programme list, compare, saved items, assistant, and institution detail are connected well.
- Data uncertainty is handled honestly. Many screens remind students to confirm with institutions, which is right for admissions.
- Compare is thoughtfully responsive: cards on mobile, a real comparison table on desktop, and shareable URL state.

**Priority Issues**

1. **[P1] Mobile nav is over-dense**
   Location: `src/components/BottomNav.jsx:71-90`, `src/components/Layout.jsx:6-14`
   Seven bottom-nav items force 9px labels and abbreviated text. This hurts discoverability and touch confidence.
   Fix: reduce primary mobile nav to 4-5 destinations, move the rest into the account/menu drawer, and use full labels.
   Suggested command: `impeccable adapt`

2. **[P1] Account drawer is not a complete accessible dialog**
   Location: `src/components/AccountDrawer.jsx:129-137`
   It has `role="dialog"` and backdrop close, but no explicit focus trap, Escape handling, or focus restoration.
   Fix: trap focus while open, close on Escape, restore focus to the opener, and mark background inert where possible.
   Suggested command: `impeccable harden`

3. **[P1] Startup splash delays access to every route**
   Location: `src/App.jsx:25-41`
   A 1450 ms enter plus 550 ms exit makes the product wait to become usable. For task UI, this feels like friction.
   Fix: only show splash on first load or while real boot work happens; shorten to a non-blocking transition.
   Suggested command: `impeccable optimize`

4. **[P1] Programme discovery exposes too many controls at once**
   Location: `src/pages/Programmes.jsx:190-343`
   Search, university, field, sort, quick-field chips, min/max points, eligibility checkbox, and clear action all compete before results.
   Fix: keep search and most-used filter visible, tuck advanced filters into an inline expandable panel, summarize active filters.
   Suggested command: `impeccable distill`

5. **[P2] Motion ignores reduced-motion in imperative scrolls**
   Location: `src/pages/Predictor.jsx:84-110`, `src/pages/Assistant.jsx:129-131`
   `scrollIntoView({ behavior: "smooth" })` runs regardless of reduced-motion preference.
   Fix: centralize a reduced-motion-aware scroll helper and use instant scroll when requested.
   Suggested command: `impeccable harden`

6. **[P2] Visual system has too many one-off materials**
   Location: `src/index.css:1`, `src/components/landing/Hero.jsx:54-63`, `src/pages/Home.jsx:121-166`, `src/pages/AuthPage.jsx:131-150`
   The app repeats cards, blur, pills, tiny uppercase labels, and Fraunces-display headings across surfaces.
   Fix: document tokens, reduce decorative blur/cards, and set stricter component rules for product vs landing surfaces.
   Suggested command: `impeccable document`, then `impeccable polish`

7. **[P2] Bundle and asset weight need a pass**
   Location: production build output, `src/lib/certificateImport.js:220-322`
   Build passes but reports a 687 kB main JS chunk, 409 kB PDF chunk, and 1.4 MB PDF worker. OCR/PDF imports are dynamic, but the main app still warns.
   Fix: inspect bundle, split rarely used routes/features, and lazy-load authenticated pages where possible.
   Suggested command: `impeccable optimize`

Alex, power user: URL filters and compare links are good, but no compact filter drawer, command-like search, or keyboard affordances slow repeat browsing.

**Minor Observations**

- `PredictorGradeSection.jsx:115` uses slate text on emerald background, which the detector flagged as washed out.
- Several controls are technically reachable but use small text and small visual affordances, especially chips and compare controls.
- The landing page now has motion, but its hero stats still resemble the banned hero-metric pattern.
- The repo lacks `PRODUCT.md` and `DESIGN.md`, so future design work has no stable source of brand/product truth.

**Questions to Consider**

- Should mobile prioritize `Home`, `Predictor`, `Programmes`, `Saved`, and `Menu`, moving Assistant/Universities/Compare into secondary navigation?
- Is Thuto meant to feel like a calm student utility or a richer admissions cockpit? The current UI is between the two.
- Should programme filtering behave more like search-first discovery, or like an advanced database browser?
